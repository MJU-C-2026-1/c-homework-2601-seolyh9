# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 주짓수 퍼포먼스 분석 및 체급 관리 시스템


## 2. 시나리오 (5~10줄)
* 이 프로그램은 사용자의 이니셜과 신체 데이터, 훈련 기록을 입력받아 체중당 스파링 효율과 훈련 종합 지수를 계산하여 출력하는 프로그램입니다.
* 사용자는 본인의 수련 기간, 현재 체중, 주간 스파링 횟수 및 훈련 시간 등을 입력합니다. 시스템은 이를 바탕으로 현재의 '성장 지수'를 계산하고, 점수 구간에 따라 관원의 등급(에이스, 유망주, 노력파 등)을 부여합니다.또한, 복합적인 조건(예: 특정 점수 이상이면서 수련 기간이 일정 개월 이상인 경우)을 만족하면 '승급 심사 대상'이나 '이달의 에이스' 칭호를 지급하는 특별 판별 로직이 포함됩니다. 최종적으로는 사용자가 메뉴판을 통해 원하는 기능을 자유롭게 선택하고, 여러 명의 관원 데이터(배열)를 동시에 관리하며 정보를 조회할 수 있도록 전체 시스템을 완성할 계획입니다.  

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
1. 관원 스탯 입력
(이니셜, 체중, 수련 기간, 주간 훈련 데이터 등 입력)
2. 성장 지수(성적) 계산
(가중치를 적용한 산술 연산을 통해 현재 퍼포먼스 수치 산출)
3. 관원 등급 산정
(성장 지수 점수 구간에 따른 등급 부여 및 출력)
4. 승급 및 칭호 지급 판별
(기간 및 점수 조건을 대조하여 승급 가능 여부 확인)
5. 관원 정보 전체 조회
(배열에 저장된 모든 관원의 데이터를 리스트 형태로 출력)
0. 프로그램 종료
(데이터 관리 시스템 종료)

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
  * 사용자로부터 메뉴의 첫 글자(이니셜), 주문 번호, 총 음식 가격, 배달비, 그리고 함께 먹은 인원수를 입력받아 1인당 최종 지불 금액을 계산하여 출력합니다.

* 데이터 특성에 따라 문자형(char - 메뉴), 정수형(int - 주문 번호), 실수형(double - 가격, 배달비, 인원)을 혼합하여 입력 화면을 구현했습니다.

* 산술 연산을 활용해 (음식 가격 + 배달비) / 인원수를 계산하고, 소수점 둘째 자리까지 정확한 정산 금액을 출력하도록 구성했습니다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** "C언어 4단계 과제 로드맵(입출력 ➔ 조건문 ➔ 반복문 ➔ 배열/함수)에 적합한 조건문과 반복문 없이 순차 구조만으로 실생활 문제를 해결할 수 있는 과제 주제 추천 요청
    * 
    * **적용 내용:** 여러 후보 중 수치 계산의 정확성을 보여줄 수 있는 '배달비 더치페이 계산기'를 선정함. 단순 합계 출력을 넘어 인원수별 분담금을 계산하는 논리적 흐름을 확립함.
      
 2. **내용 2**
    * **프롬프트 요약:** 형변환(Casting)을 배우지 않은 상태에서 정수와 실수의 나눗셈 오차를 방지하고, 필수 조건(변수 5개, 자료형 3개)을 충족하는 데이터 설계 논의
    * **적용 내용:** 강제 형변환 키워드 대신, 계산에 참여하는 주요 변수(가격, 인원 등)를 처음부터 실수형(double)으로 선언하여 자료형의 자연스러운 연산 특성을 활용함. 이를 통해 배우지 않은 문법을 지양하면서도 계산의 정확도를 높이는 구조를 모델링함.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** scanf로 메뉴 이니셜(char)을 입력받을 때, 이전 입력의 엔터값이 남아서 입력을 건너뛰는 현상이 예상됨.
     * **해결:** AI의 조언을 받아 scanf(" %c", &menu_initial);와 같이 %c 앞에 공백을 추가하여 입력 버퍼 문제를 사전에 방지함.
  2. **문제 2:** double형 변수를 입력받을 때 %f를 사용하자 값이 0으로 출력되거나 프로그램이 멈추는 현상이 발생함.
     * **해결:** 강의 자료 4주차 내용을 재확인하여, printf와 달리 scanf에서 double형은 반드시 **long float**을 의미하는 %lf 서식 지정자를 써야 함을 배우고 수정하여 해결함.
  3. **문제 3:** VS Code의 '출력(Output)' 창에서 실행 시 scanf 입력이 들어가지 않아 프로그램이 무한 대기에 빠짐.
  4. VS Code 설정에서 Run In Terminal 옵션을 활성화하거나, 터미널 탭에서 직접 명령어를 입력하여 실행함으로써 키보드 입력이 프로그램에 정상적으로 전달되도록 개선함.

## 🟩 [2차 과제: V2.0] 조건 분기 적용 (조건문)
### **✨2차 과제 업데이트 내용:**

  * 주제 전면 수정 후 1차과제 과정 반복
    수정한 주제 : 배달음식 더치페이 계산기 -> 주짓수 퍼포먼스 분석 시스템

  * 성별에 따른 체급 분류 (중첩 및 다중 조건문) : 남성(M) 과 여성(F)를 먼저 분리하고 여성 전용 체급 기준을 별도로 설정하여 시스템의 현실성을 높였습니다.

  * 승급 자격 자동 진단 (논리 연산자 &&) : '성장 지수 50점 이상' 이라는 실력적 요소와 '수련기간 6개월 이상'이라는 성실성요소를 연산자로 묵었습니다. 두 조건이 모두 참(ture)일 때만 '승급 심사 대상'이라는 메세지를 출력하여 데이터 기반의 객관적인 진단 기능을 강화했습니다. 
    
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:**  1차 과제물에 이어 2차 과제를 통한 프로그램 발전을 요청함. 
    * **적용 내용:** 조건문을 활용한 데이터 분기 처리 핵심 프로그래밍 기술 및 개념 : 다중 조건문 if ~else if else / 중첩 조건문 : if 문 안의 if문 / 논리/ 관계 연산자 :&&(AND)을 활용한 프로그램 발전. 
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 단일 체급 체계 설정으로 인해 작성자의 상황을 대입하기 어려움. 
     * **해결:** 성별에 따른 데이터 분기 처리를 통해 시스템의 정확도를 높임. 성별 입력 시 'M'이나 'F'가 아닌 다른 문자가 들어왔을 때를 대비한 else 구문을 추가하여 프로그램의 안정성을 높임.
    
  2. **문제 2:** if 조건문들을 [4] 결과 출력 영역 안에 포함해야 하는 것에 대한 의문
     * **해결:** 현재 if문 값을 계산 후 어떤 문장을 화면에 보여줄 지 결정하는 역할을 함. 이를 [4] 이전에 두려면 문자열 변수를 만들어 저장해야 하며 이때, 배열/포인터의 개념이 필요함. 따라서 4차 과제에서 해결할 것.
