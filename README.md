* 사용자는 캐릭터의 기본 능력치(체력, 마나, 공격력 등)를 입력합니다. 시스템은 이를 바탕으로 총합 전투력을 계산하고, 점수 구간에 따라 캐릭터의 전투 등급(S, A, B, C, F)을 부여합니다. 또한, 복합적인 조건(예: 특정 등급 이상이면서 마나가 일정 수치 이상인 경우)을 만족하면 '전설의 칭호'나 '보너스 골드'를 지급하는 특별 보상 로직이 포함됩니다. 최종적으로는 사용자가 메뉴판을 통해 원하는 기능을 자유롭게 선택하고, 여러 명의 캐릭터(배열)를 동시에 관리하며 정보를 조회할 수 있도록 전체 시스템을 완성할 계획입니다.

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
  * 사용자로부터 메뉴의 첫 글자(이니셜), 주문 번호, 총 음식 가격, 배달비, 그리고 함께 먹은 인원수를 입력받아 1인당 최종 지불 금액을 계산하여 출력합니다.

* 데이터 특성에 따라 문자형(char - 메뉴), 정수형(int - 주문 번호), 실수형(double - 가격, 배달비, 인원)을 혼합하여 입력 화면을 구현했습니다.

* 산술 연산을 활용해 (음식 가격 + 배달비) / 인원수를 계산하고, 소수점 둘째 자리까지 정확한 정산 금액을 출력하도록 구성했습니다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** "C언어 4단계 과제 로드맵(입출력 ➔ 조건문 ➔ 반복문 ➔ 배열/함수)에 적합한 조건문과 반복문 없이 순차 구조만으로 실생활 문제를 해결할 수 있는 과제 주제 추천 요청
    * 
    * **적용 내용:** 여러 후보 중 수치 계산의 정확성을 보여줄 수 있는 '배달비 더치페이 계산기'를 선정함. 단순 합계 출력을 넘어 인원수별 분담금을 계산하는 논리적 흐름을 확립함.
      
 2. **내용 2**
    * **프롬프트 요약:** 형변환(Casting)을 배우지 않은 상태에서 정수와 실수의 나눗셈 오차를 방지하고, 필수 조건(변수 5개, 자료형 3개)을 충족하는 데이터 설계 논의
    * **적용 내용:** 강제 형변환 키워드 대신, 계산에 참여하는 주요 변수(가격, 인원 등)를 처음부터 실수형(double)으로 선언하여 자료형의 자연스러운 연산 특성을 활용함. 이를 통해 배우지 않은 문법을 지양하면서도 계산의 정확도를 높이는 구조를 모델링함.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** scanf로 메뉴 이니셜(char)을 입력받을 때, 이전 입력의 엔터값이 남아서 입력을 건너뛰는 현상이 예상됨.
     * **해결:** AI의 조언을 받아 scanf(" %c", &menu_initial);와 같이 %c 앞에 공백을 추가하여 입력 버퍼 문제를 사전에 방지함.
  2. **문제 2:** double형 변수를 입력받을 때 %f를 사용하자 값이 0으로 출력되거나 프로그램이 멈추는 현상이 발생함.
     * **해결:** 강의 자료 4주차 내용을 재확인하여, printf와 달리 scanf에서 double형은 반드시 **long float**을 의미하는 %lf 서식 지정자를 써야 함을 배우고 수정하여 해결함.
  3. **문제 3:** VS Code의 '출력(Output)' 창에서 실행 시 scanf 입력이 들어가지 않아 프로그램이 무한 대기에 빠짐.
  4. VS Code 설정에서 Run In Terminal 옵션을 활성화하거나, 터미널 탭에서 직접 명령어를 입력하여 실행함으로써 키보드 입력이 프로그램에 정상적으로 전달되도록 개선함.
